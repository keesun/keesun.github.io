Task Repository에 제대로 등록도 했다면, 제일 먼저 이슈 목록을 보고 싶을실 겁니다.

Task List 뷰를 열면 Task Repository에 등록한 이름의 쿼리가 보이면 그것을 더블클릭하고, 안 보이면 new -> Query 메뉴를 선택하고 다음과 같이 URL을 설정해 줍니다.


Query URL와 Query Pattern은 Mylyn과 Google Code의 Issues Tracker 와 연동하기(추가)에 나와있습니다.

그럼 이제 위에서 설정한 URL을 사용해서 실제 이슈로 등록되어 있는 목록을 가져와야 하는데, 이 때 원하는 쿼리를 오른쪽 마우스로 클릭하고 싱크로나이즈 메뉴를 사용합니다.(또는 F5를 클릭합니다.)

그럼 잠시 쿼리의 이름이 깜빡이는 것을 볼 수 있습니다. 그런 다음 쿼리이름 왼쪽에 있는 화살표를 클릭하여 목록을 확인할 수 있습니다.
인코딩 문제가 있는 것을 확인할 수 있습니다. 아마 버전업이 되면 버그를 수정해서 나오지 않을까 싶습니다. 텍스트 파일에 저장할 때 String에 웹에서 긁어온 인코딩으로 인코딩 설정만 바꿔주지 않아서 이러는 것 아닐까 싶은데 정확한 원인과 해결책은 모르겠습니다.

다음에는 이슈 등록하기와 이슈 자세히 살펴보기 그리고 이슈 처리하기등을 포스팅하겠습니다.
